# ACG 发卡系统 - 部署上线指南

按以下步骤将项目发布到线上。

---

## 一、准备

- [ ] 一台 VPS 服务器（推荐 Vultr、DigitalOcean，约 $5/月）
- [ ] 一个域名（可选，也可先用 IP 访问）
- [ ] 项目已推送到 GitHub

---

## 二、部署到服务器

### 方式 A：一键脚本（推荐）

```bash
# 1. SSH 登录服务器
ssh root@你的服务器IP

# 2. 克隆项目
git clone https://github.com/你的用户名/lhm-acg-faka.git
cd lhm-acg-faka

# 3. 执行部署
chmod +x deploy.sh
./deploy.sh
```

### 方式 B：手动部署

```bash
# 1. 安装 Docker
curl -fsSL https://get.docker.com | sh

# 2. 克隆项目
git clone https://github.com/你的用户名/lhm-acg-faka.git
cd lhm-acg-faka

# 3. 启动服务（生产配置，端口 80）
docker-compose -f docker-compose.prod.yml up -d --build

# 4. 等待约 15 秒后，初始化数据库
docker exec acg-faka-web php /var/www/html/docker-init-db.php
```

---

## 三、首次配置

### 1. 修改管理员密码（必做）

```bash
docker exec acg-faka-web php /var/www/html/docker-change-admin.php 你的邮箱 新密码
```

示例：`docker exec acg-faka-web php /var/www/html/docker-change-admin.php admin@example.com MyPass123`

### 2. 配置站点域名

- 访问 `http://你的IP/admin` 登录后台
- 进入 **系统设置** → 修改 **站点域名**、**回调域名** 为你的实际域名（如 `https://faka.example.com`）

---

## 四、域名与 HTTPS（可选）

### 1. 域名解析

在 Cloudflare 或域名商添加 A 记录：

| 类型 | 主机 | 值 | 说明 |
|------|------|-----|------|
| A | @ | 服务器IP | 主域名 |
| A | www | 服务器IP | www 子域名 |

### 2. Cloudflare 配置 HTTPS

- 登录 Cloudflare → 选择站点 → SSL/TLS
- 加密模式选择 **灵活** 或 **完全**
- 开启 **始终使用 HTTPS**

### 3. 放行端口

确保服务器防火墙开放 **80**、**443** 端口。Vultr/DigitalOcean 控制台可配置。

---

## 五、验证上线

- [ ] 前台可访问：`http://你的域名` 或 `http://你的IP`
- [ ] 后台可登录：`http://你的域名/admin`
- [ ] 创建测试商品并下单
- [ ] 管理员密码已修改

---

## 六、常用命令

```bash
# 查看运行状态
docker ps

# 查看日志
docker logs acg-faka-web -f

# 停止服务
docker-compose -f docker-compose.prod.yml down

# 重启服务
docker-compose -f docker-compose.prod.yml restart
```

---

## 七、安全建议

1. **修改 MySQL 密码**：编辑 `config/database.docker.php` 和 `docker-compose.prod.yml` 中的密码
2. **定期备份**：备份 `mysql_data` 卷和 `config/` 目录
3. **防火墙**：只开放 80、443、22 端口
4. **更新**：定期 `git pull` 并 `docker-compose build --no-cache` 重新构建

---

## 故障排查

| 问题 | 处理 |
|------|------|
| 无法访问 | 检查防火墙、端口是否开放 |
| 500 错误 | 查看 `docker logs acg-faka-web` |
| 数据库连接失败 | 确认 MySQL 容器已启动 `docker ps` |
| 安装锁已存在 | 删除 `kernel/Install/Lock` 可重新安装 |
