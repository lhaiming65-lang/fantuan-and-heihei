## 二、在服务器上部署更新

### 若用 Docker 部署

```bash
# SSH 登录服务器后进入项目目录
cd lhm-acg-faka   # 或你的项目目录名

# 拉取最新代码
git pull origin main

# 重新构建并启动（使 PHP 代码生效）
docker-compose -f docker-compose.prod.yml build --no-cache
docker-compose -f docker-compose.prod.yml up -d

# 查看是否正常运行
docker ps
docker logs acg-faka-web --tail 50
```

### 若不用 Docker（PHP 直接跑）

```bash
cd 项目目录
git pull origin main
# 无需重启，PHP 下次请求即用新代码
```

## 三、上传目录权限（必做）

代码更新后，**务必保证上传目录存在且可写**，否则仍会报「文件上传失败」。

在服务器上执行（路径按你实际项目根目录改）：

```bash
# Docker 方式：在容器内创建并赋权
docker exec acg-faka-web mkdir -p /var/www/html/assets/cache/general/image
docker exec acg-faka-web chmod -R 755 /var/www/html/assets/cache

# 非 Docker：在宿主机项目根目录下
mkdir -p assets/cache/general/image
chmod -R 755 assets/cache
# 若 PHP 以 www-data 运行：
chown -R www-data:www-data assets/cache
```

## 四、部署后排查图片上传

1. **再试一次上传**  
   后台选择一张 **小于 5MB** 的图片上传。

2. **看接口返回的 `msg`**  
   打开浏览器 **F12 → Network**，找到上传请求（如 `/admin/api/upload/send?mime=image`），看响应 JSON 里的 `msg`：

   | 返回的 msg | 含义 | 处理 |
   |------------|------|------|
   | 请选择要上传的文件 | 未选文件或请求未带上 file | 检查前端是否用 `file` 字段、FormData 是否正确 |
   | 图片不能超过5MB | 文件大于 5MB | 换小图或改后端限制 |
   | 文件上传失败：临时文件无效或已过期 | 临时文件丢失/未上传 | 检查 PHP `upload_tmp_dir`、请求是否超时 |
   | 文件上传失败：无法创建上传目录，请检查权限 | 无法创建目录 | 按「三、上传目录权限」在服务器建目录、赋权 |
   | 文件上传失败：上传目录不可写，请检查权限 | 目录存在但不可写 | `chmod 755 assets/cache/general/image` 或改属主 |
   | 文件上传失败：无法写入目标目录，请检查权限或磁盘空间 | 移动文件失败 | 检查磁盘空间、SELinux、目录权限 |
   | 图片上传失败，原因：生成缩略图失败 | 缩略图生成失败 | 检查 PHP GD、图片格式是否支持 |

3. **看服务器日志**  
   ```bash
   docker logs acg-faka-web -f
   ```  
   上传时是否有 PHP 报错或 500，便于进一步定位。

## 五、PHP 配置建议（可选）

若仍怀疑是「文件过大」或「临时目录」问题，可在服务器上检查：

```bash
docker exec acg-faka-web php -i | grep -E "upload_max_filesize|post_max_size|upload_tmp_dir"
```

建议：`upload_max_filesize` 和 `post_max_size` 至少为 **6M**（大于 5MB 限制一点即可）。
